## 1. プロジェクト概要

### 1.1 目的

資産管理銀行における外国籍投資信託の管理業務を支援するシステムを構築する。海外ファンド管理会社（アドミニストレーター）から受領する基準価額（NAV）データの取込、為替換算、ファンド情報の管理、および月次運用レポートの出力を主要機能とする。

本システムはポートフォリオとしての開発であり、外国籍ファンド管理のドメイン知識、VB.NETでの開発力、および上流工程の設計力を習得することを目的とする。

### 1.2 背景

外国籍投資信託は、籍国（ルクセンブルク、ケイマン諸島、アイルランド等）の規制と日本の金融商品取引法の両方に準拠する必要があり、以下の要因から管理が複雑である。

- 基準価額（NAV）が外貨建てのため、為替換算処理が必須
- 海外と日本の営業日が異なり、データ受領タイミングにズレが生じる
- 海外アドミニストレーターからのデータ取込時に、形式チェック・業務チェックが必要
- 規制当局（金融庁等）や投資家向けに、正確かつ定型的なレポート出力が求められる

### 1.3 スコープ

#### 対象範囲

- ファンドマスタ情報の管理（登録・変更・検索・論理削除）
- 海外アドミニストレーターから受領するNAVデータのCSV取込
- 為替レートの管理（画面入力およびCSV取込）
- 営業日カレンダーの管理（日本・米国・欧州の3地域）
- 基準価額の為替換算処理（外貨建てNAV → 円建てNAV）
- 月次運用レポートのExcel出力

#### 対象外（スコープ外）

- 分配金の計算・支払処理
- 約定処理・受渡決済処理
- 投資家口座管理
- 外部システムとのリアルタイム連携（SWIFT、Bloomberg端末等）
- 規制当局向け報告書の自動生成（EDINETへの電子提出等）
- マルチテナント対応

| F-001  | ファンドマスタ管理   | 画面        | ファンド情報のCRUD、変更履歴表示、CSVエクスポート  |
| F-002  | 為替レート管理       | 画面        | 為替レートの登録・参照（画面入力＋CSV取込）        |
| F-003  | 営業日カレンダー管理 | 画面        | 日本・米国・欧州の営業日管理                       |
| F-004  | データ取込           | 画面        | NAV・為替レート・営業日カレンダーのCSV取込         |

| F-006  | 帳票出力             | 画面        | 月次運用レポートのExcel出力                        |

### 3.2 F-001：ファンドマスタ管理

#### 3.2.1 概要

外国籍投資信託の基本情報を管理する。ファンドの新規登録、編集、論理削除、検索・一覧表示、変更履歴の参照、およびCSVエクスポート機能を提供する。

#### 3.2.2 画面構成

| 画面名                 | 説明                                              |
| ---------------------- | ------------------------------------------------- |
| ファンド一覧画面       | 登録済みファンドの一覧表示・検索・CSVエクスポート |
| ファンド登録・編集画面 | ファンド情報の新規登録および編集                  |
| ファンド変更履歴画面   | 選択したファンドの変更履歴を時系列で表示          |

#### 3.2.3 管理項目

| 項目                   | 型            | 必須 | 説明                                               |
| ---------------------- | ------------- | ---- | -------------------------------------------------- |
| ISIN                   | CHAR(12)      | ○    | 国際証券コード。主キー。チェックディジット検証あり |
| ファンド名称（英語）   | NVARCHAR(200) | ○    | 英語の正式名称                                     |
| ファンド名称（日本語） | NVARCHAR(200) | ○    | 日本語の表示名称                                   |
| 通貨コード             | CHAR(3)       | ○    | ISO 4217準拠（USD, EUR, GBP等）                    |
| 籍国コード             | CHAR(2)       | ○    | ISO 3166-1 alpha-2（LU, KY, IE等）                 |
| 設定日                 | DATE          | ○    | ファンドの設定（運用開始）日                       |
| 決算頻度               | NVARCHAR(20)  | ○    | 年1回 / 年2回 / 四半期 / 毎月                      |
| 端数処理区分           | CHAR(1)       | ○    | 1:切捨て / 2:四捨五入 / 3:切上げ                   |
| ステータス             | CHAR(1)       | ○    | 1:有効 / 9:論理削除                                |
| 備考                   | NVARCHAR(500) | ―    | 自由記述                                           |

#### 3.2.4 バリデーションルール

| チェック項目 | ルール                                                                                         |
| ------------ | ---------------------------------------------------------------------------------------------- |
| ISIN形式     | 先頭2桁がアルファベット（国コード）、末尾1桁がチェックディジット（Luhnアルゴリズム）、全体12桁 |
| ISIN重複     | 同一ISINが既に登録済み（有効ステータス）でないこと                                             |
| 通貨コード   | ISO 4217の有効な通貨コードであること                                                           |
| 籍国コード   | ISO 3166-1 alpha-2の有効な国コードであること                                                   |
| 設定日       | 未来日でないこと                                                                               |

#### 3.2.5 CSVエクスポート

一覧画面の検索条件に合致するファンドをCSV形式でエクスポートする。文字コードはUTF-8（BOM付き）とし、Excelで直接開けるようにする。

#### 3.2.6 変更履歴

ファンドマスタの登録・編集・論理削除時に、変更前後の値を履歴テーブルに記録する。記録項目は以下のとおり。

- 変更日時
- 操作者（設定ファイルのユーザー名）
- 操作種別（登録 / 編集 / 削除）
- 変更対象項目名
- 変更前の値
- 変更後の値

### 3.3 F-002：為替レート管理

#### 3.3.1 概要

為替レート（TTM / TTS / TTB）を管理する。画面からの手入力とCSV取込の両方に対応する。

#### 3.3.2 画面構成

| 画面名             | 説明                                         |
| ------------------ | -------------------------------------------- |
| 為替レート一覧画面 | 通貨ペア・日付範囲を指定してレートを一覧表示 |
| 為替レート登録画面 | 特定の通貨ペア・日付のレートを手入力で登録   |

#### 3.3.3 管理項目

| 項目       | 型            | 必須 | 説明                        |
| ---------- | ------------- | ---- | --------------------------- |
| 通貨コード | CHAR(3)       | ○    | ISO 4217（USD, EUR, GBP等） |
| 基準日     | DATE          | ○    | レート公表日                |
| レート種別 | CHAR(3)       | ○    | TTM / TTS / TTB             |
| レート値   | DECIMAL(12,6) | ○    | 対円レート。小数6桁まで保持 |

#### 3.3.4 直前営業日レート参照ロジック

為替換算時に該当日のレートが存在しない場合（日本の休日等）、以下のロジックで直前営業日のレートを取得する。

1. 指定日付のレートを検索する
2. 存在しない場合、日本の営業日カレンダーを参照し、直前の営業日を特定する
3. 直前営業日のレートを取得する
4. 直前営業日にもレートがない場合、さらに遡る（最大5営業日まで）
5. 5営業日以内にレートが見つからない場合、エラーとする

### 3.4 F-003：営業日カレンダー管理

#### 3.4.1 概要

日本、米国、欧州（ルクセンブルク）の3地域の営業日カレンダーを管理する。

#### 3.4.2 画面構成

| 画面名               | 説明                                                             |
| -------------------- | ---------------------------------------------------------------- |
| 営業日カレンダー画面 | 地域・年月を指定してカレンダー形式で表示。休日の登録・削除が可能 |

#### 3.4.3 管理項目

| 項目       | 型            | 必須 | 説明                                  |
| ---------- | ------------- | ---- | ------------------------------------- |
| 地域コード | CHAR(2)       | ○    | JP:日本 / US:米国 / LU:ルクセンブルク |
| 日付       | DATE          | ○    | 対象日                                |
| 休日フラグ | BIT           | ○    | 1:休日 / 0:営業日                     |
| 休日名称   | NVARCHAR(100) | ―    | 祝日の名称（元日、MLK Day等）         |

#### 3.4.4 運用想定

- 年初に翌年分の営業日カレンダーを一括登録する（CSV取込）
- 臨時休業日の発生時は、画面から個別にメンテナンスする
- 土日は自動的に休日として扱う（DBに登録するのは祝日・臨時休業のみ）

### 3.5 F-004：データ取込

#### 3.5.1 概要

CSV形式の外部ファイルをシステムに取り込む。対象は以下の3種類。

| 取込種別         | ファイル                                             | 頻度           |
| ---------------- | ---------------------------------------------------- | -------------- |
| NAVデータ        | 海外アドミニストレーターから受領するファンド基準価額 | 日次           |
| 為替レート       | 銀行公表の為替レート（TTM/TTS/TTB）                  | 日次           |
| 営業日カレンダー | 各地域の休日一覧                                     | 年次（＋臨時） |

#### 3.5.2 画面構成

| 画面名         | 説明                                                                                |
| -------------- | ----------------------------------------------------------------------------------- |
| データ取込画面 | 取込種別を選択し、CSVファイルを指定して取り込む。取込結果（件数・エラー一覧）を表示 |

#### 3.5.3 共通バリデーション

すべての取込ファイルに対して、以下のチェックを実施する。

| チェック項目   | ルール                           |
| -------------- | -------------------------------- |
| ファイル拡張子 | .csv のみ許可                    |
| ファイルサイズ | 10MB以下                         |
| 文字コード     | UTF-8（BOM有無は問わない）       |
| ヘッダー行     | 定義済みのカラム名と一致すること |
| 空ファイル     | データ行が0件の場合はエラー      |

#### 3.5.4 NAVデータ取込

**ファイルレイアウト**

```csv
ISIN,FundName,NAVDate,Currency,NAVPerUnit,TotalNetAssets,SharesOutstanding
```

**形式チェック**

| 項目              | ルール                                       |
| ----------------- | -------------------------------------------- |
| ISIN              | 12桁、チェックディジット検証                 |
| NAVDate           | YYYY-MM-DD形式の有効な日付、未来日でないこと |
| Currency          | ISO 4217の有効な通貨コード                   |
| NAVPerUnit        | 正の数値、小数6桁以内                        |
| TotalNetAssets    | 正の数値                                     |
| SharesOutstanding | 正の数値                                     |

**業務チェック**

| 項目               | ルール                                                                 |
| ------------------ | ---------------------------------------------------------------------- |
| 重複チェック       | 同一ISIN・同一NAVDateのデータが既に登録済みでないこと                  |
| マスタ存在チェック | ISINがファンドマスタに有効ステータスで登録済みであること               |
| 異常値チェック     | 前日比でNAVPerUnitの変動率が±10%を超える場合、警告とする（取込は続行） |
| 通貨整合性チェック | 取込データのCurrencyがファンドマスタの通貨コードと一致すること         |

**取込処理方針**

- 1ファイル内の全行が形式チェックを通過した場合のみ、DB登録を実行する（オールオアナッシング）
- 業務チェックの警告（異常値検知等）は取込結果画面に表示するが、取込処理は中断しない

#### 3.5.5 為替レートCSV取込

**バリデーション**

| 項目            | ルール                                                         |

### 3.6 F-005：基準価額計算バッチ

#### 3.6.1 概要

外貨建ての基準価額（NAV）に為替レート（TTM）を乗じて、円建ての基準価額を算出する。

#### 3.6.2 実行方式

| 方式               | 説明                                                                                         |
| ------------------ | -------------------------------------------------------------------------------------------- |
| コンソールバッチ   | コマンドラインから実行。タスクスケジューラでの日次自動実行を想定                             |
| 画面からの手動実行 | 管理画面のバッチ実行ボタンから起動。対象日付を指定可能（障害復旧・データ修正時のリカバリ用） |

#### 3.6.3 処理フロー

```
入力：対象日付（指定なしの場合は当日）
　│
　├─① 対象日付の未換算NAVデータを取得（T_NAV_HISTORYからNAV_JPYがNULLのレコード）
　│
　├─② 各レコードについて：
　│    ├─ ファンドマスタから通貨コードと端数処理区分を取得
　│    ├─ 為替レートテーブルから該当日のTTMを取得
　│    │   └─ 該当日になければ直前営業日レート参照ロジック（F-002 3.3.4参照）
　│    ├─ 円換算NAV = 外貨建てNAV × TTM
　│    └─ 端数処理区分に従い端数処理を実施
　│
　├─③ 計算結果をT_NAV_HISTORYに更新（NAV_JPY, 適用レート, 計算日時）
　│
　└─④ 処理結果をログに出力（処理件数、エラー件数、所要時間）

出力：処理結果サマリ（正常件数、スキップ件数、エラー件数）
```

#### 3.6.4 計算仕様

| 項目             | 仕様                                                             |
| ---------------- | ---------------------------------------------------------------- |
| 計算式           | 円換算NAV = 外貨建てNAVPerUnit × TTM                             |
| 端数処理         | ファンドマスタの端数処理区分に従う（切捨て / 四捨五入 / 切上げ） |
| 端数処理の適用桁 | 円未満（小数点以下を処理し、整数円とする）                       |
| 使用レート       | TTM（Telegraphic Transfer Middle rate）                          |
| レート未取得時   | 直前営業日レート参照ロジックにより取得（最大5営業日遡及）        |
| レート取得不可時 | 該当ファンドをスキップし、エラーログに出力                       |

#### 3.6.5 計算例

```
Global Equity Fund（端数処理：切捨て）
  外貨建て NAV    : 52.34 USD
  適用TTM         : 150.00 JPY/USD
  円換算NAV（計算値）: 52.34 × 150.00 = 7,851.00
  円換算NAV（確定値）: 7,851 円（切捨て → 変化なし）

Euro Bond Fund（端数処理：四捨五入）
  外貨建て NAV    : 101.87 EUR
  適用TTM         : 163.45 JPY/EUR
  円換算NAV（計算値）: 101.87 × 163.45 = 16,650.6515
  円換算NAV（確定値）: 16,651 円（四捨五入）
```

### 3.7 F-006：帳票出力（月次運用レポート）

#### 3.7.1 概要

ファンドごとの月次運用レポートをExcelファイル（.xlsx）で出力する。EPPlusライブラリを使用し、グラフ・書式を含む実務レベルの帳票を生成する。

#### 3.7.2 画面構成

| 画面名       | 説明                                                              |
| ------------ | ----------------------------------------------------------------- |
| 帳票出力画面 | 対象ファンド（複数選択可）と対象年月を指定してExcelファイルを出力 |

#### 3.7.3 レポート構成

1ファンドにつき1シートを生成し、以下のセクションで構成する。

**セクション①：ファンド基本情報**

| 表示項目                      |
| ----------------------------- |
| ファンド名称（日本語 / 英語） |
| ISIN                          |
| 通貨コード                    |
| 籍国                          |
| 設定日                        |
| レポート対象年月              |

**セクション②：基準価額推移グラフ（折れ線グラフ）**

- 横軸：日付（対象月の全営業日）
- 縦軸（左）：外貨建てNAV
- 縦軸（右）：円建てNAV
- 2系列の折れ線グラフで表示

**セクション③：純資産総額推移グラフ（棒グラフ）**

- 横軸：月（直近12か月）
- 縦軸：純資産総額（外貨建て）
- 棒グラフで表示

**セクション④：リターン表**

| 期間           | 算出方法                                  |
| -------------- | ----------------------------------------- |
| 月間リターン   | (当月末NAV - 前月末NAV) ÷ 前月末NAV × 100 |
| 年初来リターン | (当月末NAV - 前年末NAV) ÷ 前年末NAV × 100 |
| 設定来リターン | (当月末NAV - 設定時NAV) ÷ 設定時NAV × 100 |

リターンは外貨建てと円建ての両方を算出する。小数第2位まで表示（第3位を四捨五入）。

**セクション⑤：適用為替レート一覧**

対象月の営業日ごとのTTM / TTS / TTBを表形式で出力する。

#### 3.7.4 出力仕様

| ファイル名         | MonthlyReport*{ISIN}*{YYYYMM}.xlsx            |
| 複数ファンド選択時 | 1ファイル内に複数シート（1ファンド＝1シート） |
| 出力先             | ユーザーが指定する任意のフォルダ              |

## 7. 制約事項・前提条件

### 7.1 前提条件

**運用面**

- 利用者は資産管理銀行の業務担当者1〜2名を想定し、シングルユーザー運用とする
- NAVデータは海外アドミニストレーターから日次でCSV形式にて提供される
- 為替レートは銀行の公表レート（TTM / TTS / TTB）を日次で取得可能とする
- 営業日カレンダーは年初に翌年分を一括登録し、臨時休業日は都度メンテナンスする

**技術面**

- 実行環境は Windows 10 以降
- ランタイムは .NET Framework 4.8（または .NET 6 以降）
- SQL Server Express 2019 以降がローカルまたはLAN上にインストール済みであること
- インターネット接続は不要（スタンドアロン運用）

### 7.2 制約事項

**ポートフォリオとしての制約**

- すべてのデータは模擬データであり、実在のファンド情報・為替レートは使用しない
- ユーザー認証・権限管理は実装しない（設定ファイルでユーザー名を固定し、監査証跡用に使用）
- 同時接続・排他制御は考慮しない（シングルユーザー前提）
- 外部システムとのリアルタイム連携（SWIFT、Bloomberg端末等）は対象外とする
- 分配金計算・約定処理・受渡決済処理は対象外とする
- マルチテナント対応は対象外とする
- データのバックアップ・リストアはSQL Serverの標準機能に委ねる

### 7.3 実運用への拡張時の考慮事項

本ポートフォリオは簡易版であるが、実運用への拡張を見据えて以下を考慮事項として記録する。

| 項目               | 拡張内容                                                                                       |
| ------------------ | ---------------------------------------------------------------------------------------------- |
| ユーザー認証       | Active Directory等の認証基盤との連携。ロールベースの権限管理（参照権限・更新権限・管理者権限） |
| 排他制御           | 同一レコードの同時更新を防止する楽観的排他制御の導入                                           |
| バッチ並列化       | 大量ファンド運用時にファンド単位で並列処理化し、スループットを向上                             |
| 為替レート自動取得 | 外部API（例：Open Exchange Rates）との連携による自動取得                                       |
| 対応地域の拡大     | アイルランド、ケイマン諸島等の営業日カレンダー追加                                             |
| 帳票の多様化       | 日次レポート、四半期レポート、規制当局向け報告書の追加                                         |

## 8. 用語集

| 用語                                                   | 説明                                                                                               |
| ------------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
| NAV（Net Asset Value）                                 | ファンドの1口あたり純資産価値（基準価額）。外貨建てNAVと円建てNAVがある                            |
| TTM（Telegraphic Transfer Middle rate）                | 銀行間の仲値（売りと買いの中間レート）。基準価額の円換算に使用                                     |
| TTS（Telegraphic Transfer Selling rate）               | 銀行が外貨を売るレート。顧客が外貨を購入する際に適用                                               |
| TTB（Telegraphic Transfer Buying rate）                | 銀行が外貨を買うレート。顧客が外貨を売却する際に適用                                               |
| ISIN（International Securities Identification Number） | 国際証券識別番号。12桁のコードでファンドを一意に識別する                                           |
| ISO 4217                                               | 通貨コードの国際規格。USD、EUR、GBP、JPY等の3文字コード                                            |
| アドミニストレーター                                   | ファンドの管理事務を行う海外の管理会社。NAVの計算・報告を担う                                      |
| カストディアン / トラスティ                            | ファンド資産の保管・管理を担う資産管理銀行                                                         |
| 端数処理                                               | 為替換算後の円建てNAV算出時に、円未満の端数を処理するルール                                        |
| 営業日カレンダー                                       | 各国・地域の銀行営業日を管理するカレンダー。休日にはNAVデータや為替レートが存在しない              |
| 論理削除                                               | レコードを物理的に削除せず、ステータスフラグで「削除済み」とする方式。監査証跡を保持するために採用 |
| 監査証跡（Audit Trail）                                | 誰が・いつ・何を変更したかを記録する証跡。金融システムでは規制上必須                               |
| EPPlus                                                 | .NET環境でExcelファイルを生成するOSSライブラリ                                                     |
| NLog                                                   | .NET環境のログ出力ライブラリ。ファイル出力・ログレベル制御・日次ローテーションに対応               |
