## 1. バッチ概要

### 1.1 バッチ一覧

本システムで提供するバッチは以下の1本である。

| バッチID | バッチ名 | 実行方式 | 概要 |
|---------|---------|---------|------|
| BAT-001 | 基準価額計算バッチ | コンソールアプリ + 画面手動実行 | 外貨建てNAVに為替レート（TTM）を乗じて円建てNAVを算出する |

### 1.2 実行方式

| 方式 | 説明 | 用途 |
|------|------|------|
| コンソール実行 | コマンドラインから引数指定で実行 | タスクスケジューラによる日次自動実行 |
| 画面手動実行 | SCR-008 のボタンから起動 | 障害復旧・データ修正時のリカバリ、日次確認 |

両方式とも同一の NavCalculationService.Execute() を呼び出す。処理ロジックに差異はない。

## 2. BAT-001：基準価額計算バッチ

### 2.1 入出力定義

**入力**

| # | 入力元 | 内容 |
|---|--------|------|
| 1 | コマンドライン引数 | 対象日付（YYYY-MM-DD形式）。省略時は当日日付 |
| 2 | T_NAV_HISTORY | 対象日付の未換算NAVデータ（NAV_JPY が NULL のレコード） |
| 3 | M_FUND | ファンドマスタ（通貨コード、端数処理区分） |
| 4 | M_EXCHANGE_RATE | 為替レート（TTM） |
| 5 | M_CALENDAR | 営業日カレンダー（直前営業日判定用） |

**出力**

| # | 出力先 | 内容 |
|---|--------|------|
| 1 | T_NAV_HISTORY | 円建てNAV、適用レート、計算日時の更新 |
| 2 | ログファイル | 処理結果サマリ、各レコードの計算結果、エラー・警告 |
| 3 | 標準出力（コンソール） | 処理結果サマリ（コンソール実行時） |

### 2.2 コマンドライン仕様

```
ForeignFundManager.Batch.exe [/date:YYYY-MM-DD]

  /date   対象日付を指定する。省略時は当日日付。
          未来日を指定した場合はエラー終了（終了コード 9）。

  例：
    ForeignFundManager.Batch.exe                     → 当日分を処理
    ForeignFundManager.Batch.exe /date:2025-01-15    → 2025/01/15分を処理
```

**終了コード**

| コード | 意味 | 条件 |
|--------|------|------|
| 0 | 正常終了 | 全件処理成功 |
| 1 | 警告終了 | 一部スキップあり（レート取得不可等）だが、処理自体は完了 |
| 9 | 異常終了 | DB接続失敗、引数不正等のシステムエラー |

### 2.3 処理フロー図

```
START
  |
  v
[1] Parse command line arguments
  |-- Invalid -> LOG FATAL, EXIT(9)
  |
  v
[2] Connect to database
  |-- Failed -> LOG FATAL, EXIT(9)
  |
  v
[3] Get unconverted NAV records
  |   SELECT from T_NAV_HISTORY
  |   WHERE NAV_DATE = @targetDate AND NAV_JPY IS NULL
  |
  |-- 0 records -> LOG INFO "No records to process", EXIT(0)
  |
  v
[4] FOR EACH NAV record
  |
  |   [4-1] Get fund master (M_FUND)
  |     |-- Not found -> LOG ERROR, mark as ERROR, CONTINUE
  |     |
  |     v
  |   [4-2] Get TTM rate (M_EXCHANGE_RATE)
  |     |-- Found for target date -> use it
  |     |-- Not found -> [4-2a] Previous business day lookup
  |     |     |-- Search M_CALENDAR + M_EXCHANGE_RATE
  |     |     |-- Go back up to 5 business days
  |     |     |-- Found -> use it, LOG WARN "Previous day rate used"
  |     |     |-- Not found within 5 days -> LOG ERROR, mark as SKIP, CONTINUE
  |     |
  |     v
  |   [4-3] Calculate NAV_JPY
  |     |   NAV_JPY = NAV_PER_UNIT * TTM
  |     |   Apply rounding (per fund's ROUNDING_TYPE)
  |     |
  |     v
  |   [4-4] Update T_NAV_HISTORY
  |     |   SET NAV_JPY, APPLIED_RATE, CALCULATED_AT, UPDATED_AT, UPDATED_BY
  |     |
  |     v
  |   [4-5] LOG INFO "Calculated: ISIN=xxx, NAV_JPY=xxx"
  |
  v (END FOR EACH)
[5] Output summary
  |   Total / Success / Skipped / Error counts
  |   Elapsed time
  |
  v
[6] Determine exit code
  |-- All success -> EXIT(0)
  |-- Has skipped or error -> EXIT(1)
  |
  v
END
```

### 2.4 計算仕様

#### 計算式

```
円換算NAV = 外貨建てNAVPerUnit × TTM（仲値）
```

#### 端数処理

ファンドマスタの ROUNDING_TYPE に従い、円未満の端数を処理する。

| ROUNDING_TYPE | 処理 | VB.NET での実装 |
|--------------|------|----------------|
| 1（切捨て） | 小数点以下を切り捨て | Math.Floor(value) |
| 2（四捨五入） | 小数点以下を四捨五入 | Math.Round(value, 0, MidpointRounding.AwayFromZero) |
| 3（切上げ） | 小数点以下を切り上げ | Math.Ceiling(value) |

> **注意：** VB.NET の Math.Round は既定で「銀行家の丸め（Banker's Rounding）」を使用する。金融業務で一般的な「四捨五入」（0.5 を常に切り上げ）を実現するには、MidpointRounding.AwayFromZero を明示指定する必要がある。

#### 計算例

```
Example 1: Global Equity Fund (ROUNDING_TYPE = 1: Truncate)
  NAV_PER_UNIT : 52.34 USD
  TTM          : 150.00 JPY/USD
  Calculated   : 52.34 * 150.00 = 7,851.00
  NAV_JPY      : 7,851 (no fractional part)

Example 2: Euro Bond Fund (ROUNDING_TYPE = 2: Round)
  NAV_PER_UNIT : 101.87 EUR
  TTM          : 163.45 JPY/EUR
  Calculated   : 101.87 * 163.45 = 16,650.6515
  NAV_JPY      : 16,651 (rounded up)

Example 3: Asia Pacific Fund (ROUNDING_TYPE = 3: Ceiling)
  NAV_PER_UNIT : 28.91 USD
  TTM          : 150.00 JPY/USD
  Calculated   : 28.91 * 150.00 = 4,336.50
  NAV_JPY      : 4,337 (ceiling)
```

### 2.5 直前営業日レート参照ロジック

為替換算時に該当日のTTMが存在しない場合に、直前の営業日のレートを取得するロジック。

```
Input: targetDate, currencyCode
Output: TTM rate (or NULL if not found)

1. rate = GetRate(currencyCode, targetDate)
2. IF rate IS NOT NULL THEN RETURN rate

3. lookbackCount = 0
4. currentDate = targetDate

5. WHILE lookbackCount < 5
     currentDate = currentDate - 1 day

     IF currentDate is Saturday or Sunday THEN
       CONTINUE  (skip weekends, do not count)

     IF EXISTS in M_CALENDAR
       WHERE REGION_CODE = 'JP'
       AND CALENDAR_DATE = currentDate
       AND IS_HOLIDAY = 1 THEN
       CONTINUE  (skip JP holiday, do not count)

     lookbackCount = lookbackCount + 1
     rate = GetRate(currencyCode, currentDate)
     IF rate IS NOT NULL THEN
       LOG WARN "Previous business day rate used: {currentDate}"
       RETURN rate

6. RETURN NULL  (not found within 5 business days)
```

> **カウント方式：** 「最大5営業日」とは、土日・日本の祝日を除いた実営業日を5日分遡ることを意味する。カレンダー上の暦日ではない。

### 2.6 トランザクション方針

| 方針 | 説明 |
|------|------|
| トランザクション範囲 | 1レコード（1ファンド）の更新ごとにコミットする |
| 理由 | 1件のエラーが他のファンドの処理に影響しないようにするため |
| ロールバック | 個別レコードの更新失敗時は当該レコードのみロールバックし、次のレコードの処理を継続する |

> **画面操作（マスタ登録等）では「オールオアナッシング」方式だが、バッチでは「個別コミット」方式を採用する。** これは、10ファンド中1ファンドだけレート未取得で失敗した場合に、残り9ファンドの計算結果を無効にすべきでないためである。

### 2.8 リカバリ運用

| シナリオ | 対処方法 |
|---------|---------|
| バッチが途中で異常終了した | 同一日付で再実行する。NAV_JPY が NULL のレコードのみ処理対象となるため、計算済みのレコードは二重処理されない |
| 誤ったレートで計算してしまった | ① T_NAV_HISTORY の NAV_JPY / APPLIED_RATE / CALCULATED_AT を NULL に戻す → ② 正しいレートを登録 → ③ 同一日付で再実行 |
| 過去日付の計算漏れが発覚した | 画面（SCR-008）から対象日付を指定して手動実行する |
