## 2. 共通仕様

### 2.1 CSV共通仕様（入力ファイル）

| 項目 | 仕様 |
|------|------|
| 文字コード | UTF-8（BOM有無は問わない） |
| 改行コード | CRLF または LF |
| 区切り文字 | カンマ（,） |
| 囲み文字 | ダブルクォーテーション（"）。値にカンマを含む場合は必須 |
| ヘッダー行 | 1行目。カラム名は定義済みの名称と完全一致すること |
| 空行 | 無視する |
| ファイル拡張子 | .csv のみ許可 |
| 最大ファイルサイズ | 10MB |

### 2.2 CSV共通仕様（出力ファイル）

| 項目 | 仕様 |
|------|------|
| 文字コード | UTF-8（BOM付き） |
| 改行コード | CRLF |
| 区切り文字 | カンマ（,） |
| 囲み文字 | 値にカンマまたはダブルクォーテーションを含む場合のみ付与 |
| ヘッダー行 | 1行目に出力 |

> **BOM付きUTF-8の理由：** ExcelでCSVファイルを直接開いた場合、BOMなしUTF-8では日本語が文字化けする。業務担当者がExcelで確認することを想定し、BOM付きとする。

### 2.3 共通バリデーション

すべての入力CSVファイルに対して実施する前提チェック。

| # | チェック項目 | ルール | エラー時の動作 |
|---|------------|--------|-------------|
| 1 | ファイル存在 | 指定パスにファイルが存在すること | エラー（取込中止） |
| 2 | ファイル拡張子 | .csv であること | エラー（取込中止） |
| 3 | ファイルサイズ | 10MB以下であること | エラー（取込中止） |
| 4 | ヘッダー行 | 定義済みカラム名と完全一致すること | エラー（取込中止） |
| 5 | データ行数 | 1行以上であること（空ファイル禁止） | エラー（取込中止） |
| 6 | 文字コード | UTF-8としてデコード可能であること | エラー（取込中止） |

### 2.4 取込処理方針

| 方針 | 説明 |
|------|------|
| オールオアナッシング | 1ファイル内の全行が形式チェックを通過した場合のみ、DB登録を実行する |
| 形式チェック失敗時 | 全行の形式チェック結果をまとめてエラー一覧として表示。1件もDBに登録しない |
| 業務チェック（警告） | 取込は続行する。警告内容を T_IMPORT_LOG_DETAIL に記録し、取込結果画面に表示する |
| トランザクション | 1ファイルの全行INSERT を1トランザクションで実行する |

## 3. IF-IN-001：NAVデータCSV

### 3.1 ファイルレイアウト

| # | カラム名 | 型 | 必須 | 桁数 | 説明 |
|---|---------|-----|------|------|------|
| 1 | ISIN | 文字列 | ○ | 12桁固定 | 国際証券識別番号 |
| 2 | FundName | 文字列 | ○ | 最大200桁 | ファンド名称（英語）。取込時はバリデーション参照用のみ |
| 3 | NAVDate | 日付 | ○ | YYYY-MM-DD | NAV算出日 |
| 4 | Currency | 文字列 | ○ | 3桁固定 | 通貨コード（ISO 4217） |
| 5 | NAVPerUnit | 数値 | ○ | 整数12桁.小数6桁 | 1口あたり基準価額（外貨建て） |
| 6 | TotalNetAssets | 数値 | ○ | 整数16桁.小数2桁 | 純資産総額 |
| 7 | SharesOutstanding | 数値 | ○ | 整数16桁.小数2桁 | 発行済口数 |

### 3.2 サンプルデータ

```csv
ISIN,FundName,NAVDate,Currency,NAVPerUnit,TotalNetAssets,SharesOutstanding
LU0123456789,Global Equity Fund,2025-01-15,USD,52.340000,1200000000.00,22920519.68
IE00ABCDEF01,Euro Bond Fund,2025-01-15,EUR,101.870000,800000000.00,7853198.39
KY9876543210,Asia Pacific Fund,2025-01-15,USD,28.910000,500000000.00,17296436.18
```

### 3.3 バリデーション詳細

#### 形式チェック（1行でもNGなら全件取込中止）

| # | 対象カラム | チェック内容 | エラーメッセージ |
|---|-----------|------------|----------------|
| 1 | ISIN | 12桁であること | 「行{N}: ISINは12桁で入力してください（値: {value}）」 |
| 2 | ISIN | チェックディジットが正しいこと（Luhnアルゴリズム） | 「行{N}: ISINのチェックディジットが不正です（値: {value}）」 |
| 3 | NAVDate | YYYY-MM-DD形式の有効な日付であること | 「行{N}: NAVDateが不正な日付形式です（値: {value}）」 |
| 4 | NAVDate | 未来日でないこと | 「行{N}: NAVDateに未来日は指定できません（値: {value}）」 |
| 5 | Currency | ISO 4217の有効な3桁コードであること | 「行{N}: Currencyが不正です（値: {value}）」 |
| 6 | NAVPerUnit | 正の数値であること | 「行{N}: NAVPerUnitは正の数値で入力してください（値: {value}）」 |
| 7 | NAVPerUnit | 小数6桁以内であること | 「行{N}: NAVPerUnitの小数桁数が上限を超えています（値: {value}）」 |
| 8 | TotalNetAssets | 正の数値であること | 「行{N}: TotalNetAssetsは正の数値で入力してください（値: {value}）」 |
| 9 | SharesOutstanding | 正の数値であること | 「行{N}: SharesOutstandingは正の数値で入力してください（値: {value}）」 |

#### 業務チェック

| # | チェック内容 | 種別 | 動作 |
|---|------------|------|------|
| 1 | 同一ISIN・同一NAVDateのデータがT_NAV_HISTORYに存在しないこと | エラー | 取込中止 |
| 2 | ISINがM_FUNDに有効ステータス（STATUS='1'）で登録済みであること | エラー | 取込中止 |
| 3 | CurrencyがM_FUNDの通貨コードと一致すること | エラー | 取込中止 |
| 4 | 前日比でNAVPerUnitの変動率が±10%を超えていないこと | 警告 | 取込続行 |

#### 変動率チェックの計算式

```
前日NAV = T_NAV_HISTORYから同一ISINの直近NAVPerUnitを取得
変動率(%) = |当日NAV - 前日NAV| ÷ 前日NAV × 100

変動率 > 10% の場合、警告を出力する。
前日NAVが存在しない場合（初回取込等）、変動率チェックはスキップする。
```

### 3.4 取込後の格納先

| CSVカラム | 格納先テーブル.カラム | 変換 |
|----------|-------------------|------|
| ISIN | T_NAV_HISTORY.ISIN | そのまま |
| FundName | （格納しない） | 参照用のみ |
| NAVDate | T_NAV_HISTORY.NAV_DATE | DATE型に変換 |
| Currency | T_NAV_HISTORY.CURRENCY_CODE | そのまま |
| NAVPerUnit | T_NAV_HISTORY.NAV_PER_UNIT | DECIMAL(18,6)に変換 |
| TotalNetAssets | T_NAV_HISTORY.TOTAL_NET_ASSETS | DECIMAL(18,2)に変換 |
| SharesOutstanding | T_NAV_HISTORY.SHARES_OUTSTANDING | DECIMAL(18,2)に変換 |
| ― | T_NAV_HISTORY.NAV_JPY | NULL（バッチ処理で後から更新） |
| ― | T_NAV_HISTORY.APPLIED_RATE | NULL（バッチ処理で後から更新） |
| ― | T_NAV_HISTORY.CALCULATED_AT | NULL（バッチ処理で後から更新） |
| ― | T_NAV_HISTORY.IMPORT_LOG_ID | 当回のT_IMPORT_LOG.IMPORT_LOG_ID |

## 4. IF-IN-002：為替レートCSV

### 4.1 ファイルレイアウト

| # | カラム名 | 型 | 必須 | 桁数 | 説明 |
|---|---------|-----|------|------|------|
| 1 | CurrencyCode | 文字列 | ○ | 3桁固定 | 通貨コード（ISO 4217） |
| 2 | RateDate | 日付 | ○ | YYYY-MM-DD | レート公表日 |
| 3 | TTM | 数値 | ○ | 整数6桁.小数6桁 | 仲値（対円） |
| 4 | TTS | 数値 | ○ | 整数6桁.小数6桁 | 売レート（対円） |
| 5 | TTB | 数値 | ○ | 整数6桁.小数6桁 | 買レート（対円） |

### 4.2 サンプルデータ

```csv
CurrencyCode,RateDate,TTM,TTS,TTB
USD,2025-01-15,150.000000,151.000000,149.000000
EUR,2025-01-15,163.450000,164.450000,162.450000
GBP,2025-01-15,190.000000,191.000000,189.000000
```

### 4.3 バリデーション詳細

#### 形式チェック

| # | 対象カラム | チェック内容 | エラーメッセージ |
|---|-----------|------------|----------------|
| 1 | CurrencyCode | M_CURRENCYに存在する有効な通貨コードであること | 「行{N}: CurrencyCodeが不正です（値: {value}）」 |
| 2 | RateDate | YYYY-MM-DD形式の有効な日付であること | 「行{N}: RateDateが不正な日付形式です（値: {value}）」 |
| 3 | RateDate | 未来日でないこと | 「行{N}: RateDateに未来日は指定できません（値: {value}）」 |
| 4 | TTM | 正の数値であること | 「行{N}: TTMは正の数値で入力してください（値: {value}）」 |
| 5 | TTS | 正の数値であること | 「行{N}: TTSは正の数値で入力してください（値: {value}）」 |
| 6 | TTB | 正の数値であること | 「行{N}: TTBは正の数値で入力してください（値: {value}）」 |
| 7 | TTM/TTS/TTB | 小数6桁以内であること | 「行{N}: レート値の小数桁数が上限を超えています」 |

#### 業務チェック

| # | チェック内容 | 種別 | 動作 |
|---|------------|------|------|
| 1 | TTB ≤ TTM ≤ TTS であること | エラー | 取込中止 |
| 2 | 同一CurrencyCode・同一RateDateが既に登録済みでないこと | エラー | 取込中止 |

### 4.4 取込後の格納先

| CSVカラム | 格納先テーブル.カラム |
|----------|-------------------|
| CurrencyCode | M_EXCHANGE_RATE.CURRENCY_CODE |
| RateDate | M_EXCHANGE_RATE.RATE_DATE |
| TTM | M_EXCHANGE_RATE.TTM |
| TTS | M_EXCHANGE_RATE.TTS |
| TTB | M_EXCHANGE_RATE.TTB |
