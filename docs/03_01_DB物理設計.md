## 1. 物理設計方針

### 1.1 本書の位置づけ

DB論理設計書（BD-001）で定義したテーブル構造を、SQL Server Express 2019 上で実行可能なDDLとして具体化する。論理設計からの主な追加事項は以下の通り。

| 項目 | 内容 |
|------|------|
| データベース名 | ForeignFundManager |
| 照合順序 | Japanese_CI_AS（日本語大文字小文字区別なし・アクセント区別あり） |
| スキーマ | dbo（既定スキーマ） |
| ファイルグループ | PRIMARY（既定。Express Edition のためファイルグループ分割は行わない） |
| リカバリモデル | SIMPLE（開発・デモ用途のため。本番ではFULLを推奨） |

### 1.2 DDLスクリプト構成

実行順序に従い、以下のスクリプトファイルに分割する。

| # | ファイル名 | 内容 | 冪等性 |
|---|-----------|------|--------|
| 1 | 01_CREATE_DATABASE.sql | データベース作成 | IF NOT EXISTS で冪等 |
| 2 | 02_CREATE_TABLES.sql | 全テーブルの CREATE TABLE（PK・FK・CHECK制約含む） | DROP IF EXISTS → CREATE |
| 3 | 03_CREATE_INDEX.sql | 非クラスタ化インデックスの作成 | DROP IF EXISTS → CREATE |
| 4 | 04_INSERT_MASTER.sql | マスタ初期データ投入（通貨・国・コードマスタ） | MERGE 方式で冪等 |
| 5 | 05_INSERT_SAMPLE_DATA.sql | デモ用模擬データ投入（ファンド・NAV・為替レート等） | DELETE → INSERT（FK制約によりTRUNCATE不可のため） |

### 1.3 論理設計からの変更点

| 項目 | 論理設計（BD-001） | 物理設計（本書） | 変更理由 |
|------|-------------------|-----------------|---------|
| FK の ON DELETE | RESTRICT | NO ACTION | SQL Server では RESTRICT キーワードが使用不可。NO ACTION が同等の動作 |
| FK の ON UPDATE | RESTRICT | NO ACTION | 同上 |
| T_IMPORT_LOG_DETAIL の ON DELETE | CASCADE | CASCADE | 論理設計通り。明細は親ログと連動削除 |
| DEFAULT制約 | 論理設計では暗黙 | 明示的に DEFAULT 制約を付与 | DDLの明確化。STATUS, IS_ACTIVE, 件数カラム等 |

### 1.4 テーブル作成順序

外部キー制約の依存関係に基づき、以下の順序でテーブルを作成する。

```
1. M_CURRENCY     （依存先なし）
2. M_COUNTRY      （依存先なし）
3. M_CODE         （依存先なし）
4. M_FUND         （→ M_CURRENCY, M_COUNTRY）
5. M_EXCHANGE_RATE（→ M_CURRENCY）
6. M_CALENDAR     （依存先なし ※順序は任意だが、マスタをまとめて先に作成）
7. T_IMPORT_LOG   （依存先なし）
8. T_NAV_HISTORY  （→ M_FUND, M_CURRENCY, T_IMPORT_LOG）
9. T_AUDIT_LOG    （依存先なし ※トランザクション系としてまとめる）
10. T_IMPORT_LOG_DETAIL（→ T_IMPORT_LOG）
```

| 物理設計補足 | 内容 |

### 2.4 M_FUND（ファンドマスタ）

**対応する論理設計：** BD-001 §3.1

```sql
CREATE TABLE [dbo].[M_FUND] (
    [ISIN]                   CHAR(12)       NOT NULL,
    [FUND_NAME_EN]           NVARCHAR(200)  NOT NULL,
    [FUND_NAME_JP]           NVARCHAR(200)  NOT NULL,
    [CURRENCY_CODE]          CHAR(3)        NOT NULL,
    [DOMICILE_COUNTRY_CODE]  CHAR(2)        NOT NULL,
    [INCEPTION_DATE]         DATE           NOT NULL,
    [SETTLEMENT_FREQUENCY]   NVARCHAR(20)   NOT NULL,
    [ROUNDING_TYPE]          CHAR(1)        NOT NULL,
    [STATUS]                 CHAR(1)        NOT NULL  CONSTRAINT [DF_M_FUND_STATUS] DEFAULT ('1'),
    [REMARKS]                NVARCHAR(500)  NULL,
    [CREATED_AT]             DATETIME       NOT NULL,
    [CREATED_BY]             NVARCHAR(50)   NOT NULL,
    [UPDATED_AT]             DATETIME       NOT NULL,
    [UPDATED_BY]             NVARCHAR(50)   NOT NULL,
    CONSTRAINT [PK_M_FUND]            PRIMARY KEY CLUSTERED ([ISIN]),
    CONSTRAINT [FK_M_FUND_M_CURRENCY] FOREIGN KEY ([CURRENCY_CODE])
        REFERENCES [dbo].[M_CURRENCY] ([CURRENCY_CODE]) ON DELETE NO ACTION ON UPDATE NO ACTION,
    CONSTRAINT [FK_M_FUND_M_COUNTRY]  FOREIGN KEY ([DOMICILE_COUNTRY_CODE])
        REFERENCES [dbo].[M_COUNTRY] ([COUNTRY_CODE]) ON DELETE NO ACTION ON UPDATE NO ACTION
);
```

| 物理設計補足 | 内容 |
|-------------|------|
| クラスタ化インデックス | PK_M_FUND（ISIN） |
| 想定レコード数 | 5〜10件（デモ用） |
| 更新頻度 | 低（マスタ登録・編集時） |
| DEFAULT制約 | STATUS='1'（有効）を明示 |
| 論理削除 | STATUS='9' で論理削除。物理削除は行わない |

### 2.5 M_EXCHANGE_RATE（為替レートマスタ）

**対応する論理設計：** BD-001 §3.4

```sql
CREATE TABLE [dbo].[M_EXCHANGE_RATE] (
    [CURRENCY_CODE] CHAR(3)        NOT NULL,
    [RATE_DATE]     DATE           NOT NULL,
    [TTM]           DECIMAL(12,6)  NOT NULL,
    [TTS]           DECIMAL(12,6)  NOT NULL,
    [TTB]           DECIMAL(12,6)  NOT NULL,
    [CREATED_AT]    DATETIME       NOT NULL,
    [CREATED_BY]    NVARCHAR(50)   NOT NULL,
    [UPDATED_AT]    DATETIME       NOT NULL,
    [UPDATED_BY]    NVARCHAR(50)   NOT NULL,
    CONSTRAINT [PK_M_EXCHANGE_RATE]           PRIMARY KEY CLUSTERED ([CURRENCY_CODE], [RATE_DATE]),
    CONSTRAINT [FK_M_EXCHANGE_RATE_M_CURRENCY] FOREIGN KEY ([CURRENCY_CODE])
        REFERENCES [dbo].[M_CURRENCY] ([CURRENCY_CODE]) ON DELETE NO ACTION ON UPDATE NO ACTION,
    CONSTRAINT [CK_M_EXCHANGE_RATE_ORDER]     CHECK ([TTB] <= [TTM] AND [TTM] <= [TTS])
);
```

| 物理設計補足 | 内容 |
|-------------|------|
| クラスタ化インデックス | PK_M_EXCHANGE_RATE（CURRENCY_CODE, RATE_DATE） |
| 想定レコード数 | 通貨数 × 営業日数（3通貨 × 250日 = 750件/年） |
| 更新頻度 | 日次（為替レートCSV取込 or 手入力） |
| CHECK制約 | TTB ≤ TTM ≤ TTS のレート大小関係を保証 |
| DECIMAL精度 | (12,6)：整数6桁.小数6桁。為替レートの精度を十分確保 |

### 2.8 T_NAV_HISTORY（NAV履歴）

**対応する論理設計：** BD-001 §3.7

```sql
CREATE TABLE [dbo].[T_NAV_HISTORY] (
    [NAV_HISTORY_ID]    INT            IDENTITY(1,1)  NOT NULL,
    [ISIN]              CHAR(12)       NOT NULL,
    [NAV_DATE]          DATE           NOT NULL,
    [CURRENCY_CODE]     CHAR(3)        NOT NULL,
    [NAV_PER_UNIT]      DECIMAL(18,6)  NOT NULL,
    [NAV_JPY]           DECIMAL(18,0)  NULL,
    [TOTAL_NET_ASSETS]  DECIMAL(18,2)  NOT NULL,
    [SHARES_OUTSTANDING] DECIMAL(18,2) NOT NULL,
    [APPLIED_RATE]      DECIMAL(12,6)  NULL,
    [CALCULATED_AT]     DATETIME       NULL,
    [IMPORT_LOG_ID]     INT            NOT NULL,
    [CREATED_AT]        DATETIME       NOT NULL,
    [CREATED_BY]        NVARCHAR(50)   NOT NULL,
    [UPDATED_AT]        DATETIME       NOT NULL,
    [UPDATED_BY]        NVARCHAR(50)   NOT NULL,
    CONSTRAINT [PK_T_NAV_HISTORY]             PRIMARY KEY CLUSTERED ([NAV_HISTORY_ID]),
    CONSTRAINT [UQ_T_NAV_HISTORY_ISIN_DATE]   UNIQUE ([ISIN], [NAV_DATE]),
    CONSTRAINT [FK_T_NAV_HISTORY_M_FUND]      FOREIGN KEY ([ISIN])
        REFERENCES [dbo].[M_FUND] ([ISIN]) ON DELETE NO ACTION ON UPDATE NO ACTION,
    CONSTRAINT [FK_T_NAV_HISTORY_M_CURRENCY]  FOREIGN KEY ([CURRENCY_CODE])
        REFERENCES [dbo].[M_CURRENCY] ([CURRENCY_CODE]) ON DELETE NO ACTION ON UPDATE NO ACTION,
    CONSTRAINT [FK_T_NAV_HISTORY_T_IMPORT_LOG] FOREIGN KEY ([IMPORT_LOG_ID])
        REFERENCES [dbo].[T_IMPORT_LOG] ([IMPORT_LOG_ID]) ON DELETE NO ACTION ON UPDATE NO ACTION
);
```

| 物理設計補足 | 内容 |
|-------------|------|
| クラスタ化インデックス | PK_T_NAV_HISTORY（NAV_HISTORY_ID、IDENTITY自動採番） |
| ユニーク制約 | UQ_T_NAV_HISTORY_ISIN_DATE（同一ファンド・同一日のNAV重複を防止） |
| 想定レコード数 | ファンド数 × 営業日数（10ファンド × 250日 = 2,500件/年） |
| NULL許可カラム | NAV_JPY, APPLIED_RATE, CALCULATED_AT はバッチ処理前はNULL |
| DECIMAL精度 | NAV_PER_UNIT(18,6)：小数6桁で外貨建てNAVの精度を確保。NAV_JPY(18,0)：円建ては整数 |

## 3. インデックス物理定義

### 3.1 非クラスタ化インデックス一覧

| # | テーブル | インデックス名 | カラム | 目的 |
|---|---------|--------------|--------|------|

| 6 | T_IMPORT_LOG_DETAIL | IX_T_IMPORT_LOG_DETAIL_LOG_ID | IMPORT_LOG_ID | 取込明細の親ログ別取得 |

### 5.1 M_FUND（ファンドマスタ）

| LU9876543210 | Euro Bond Fund | ユーロ債券ファンド | EUR | LU | 2（四捨五入） |
| KY1234567890 | Asia Pacific Growth Fund | アジアパシフィック成長ファンド | USD | KY | 1（切捨て） |

| LU1111222233 | Swiss Multi-Asset Fund | スイス・マルチアセットファンド | CHF | CH | 2（四捨五入） |

> **ISINコードについて：** 上記は模擬コードであり、実在のファンドとは無関係。チェックディジットの検証は実装時にバリデータで対応するが、模擬データ投入時は検証をスキップする。

### 5.2 M_EXCHANGE_RATE（為替レート）

2025年1月〜3月の日本（JP）・米国（US）・ルクセンブルク（LU）の祝日を登録する。

### 5.4 T_NAV_HISTORY / T_IMPORT_LOG

2025年1月の営業日分のNAVデータおよび取込ログの模擬データを投入する。一部は円換算済み（バッチ処理済み）とし、一部は未計算（NAV_JPY = NULL）の状態とする。これにより、デモ時にバッチ実行の動作確認が可能となる。
