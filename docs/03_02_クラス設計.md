## 4. Repositories 詳細

全リポジトリの共通設計方針：
- SqlConnection / SqlTransaction は呼び出し元（Service層）から受け取る
- パラメータ化クエリを必ず使用（SQLインジェクション対策）
- SELECT系はモデルを返却、INSERT/UPDATE/DELETEは影響行数（Integer）を返却
- 結果セットのマッピングはSqlDataReaderからプロパティに手動マッピング

### 4.1 FundRepository

```vb
Public Class FundRepository

    ''' <summary>検索条件に合致するファンド一覧を取得</summary>
    ''' <param name="isin">ISIN前方一致（空=条件なし）</param>
    ''' <param name="fundName">名称部分一致（英語名 OR 日本語名、空=条件なし）</param>
    ''' <param name="currencyCode">通貨コード（空=全て）</param>
    ''' <param name="countryCode">籍国コード（空=全て）</param>
    ''' <param name="activeOnly">True=STATUS='1'のみ、False=全て</param>
    Public Function Search(isin As String, fundName As String,
                           currencyCode As String, countryCode As String,
                           activeOnly As Boolean,
                           conn As SqlConnection,
                           Optional trans As SqlTransaction = Nothing) As List(Of Fund)
    ' SQL:
    '   SELECT * FROM M_FUND
    '   WHERE (@isin = '' OR ISIN LIKE @isin + '%')
    '     AND (@name = '' OR FUND_NAME_EN LIKE '%' + @name + '%'
    '                     OR FUND_NAME_JP LIKE '%' + @name + '%')
    '     AND (@ccy = '' OR CURRENCY_CODE = @ccy)
    '     AND (@ctry = '' OR DOMICILE_COUNTRY_CODE = @ctry)
    '     AND (@activeOnly = 0 OR STATUS = '1')
    '   ORDER BY ISIN

    ''' <summary>ISINでファンド1件を取得</summary>
    Public Function GetByIsin(isin As String, conn As SqlConnection,
                               Optional trans As SqlTransaction = Nothing) As Fund
    ' SQL: SELECT * FROM M_FUND WHERE ISIN = @isin

    ''' <summary>ISINの存在チェック</summary>
    Public Function Exists(isin As String, conn As SqlConnection,
                            Optional trans As SqlTransaction = Nothing) As Boolean
    ' SQL: SELECT COUNT(*) FROM M_FUND WHERE ISIN = @isin

    ''' <summary>新規登録</summary>
    Public Function Insert(fund As Fund, conn As SqlConnection,
                            trans As SqlTransaction) As Integer
    ' SQL: INSERT INTO M_FUND (全カラム) VALUES (...)

    ''' <summary>更新</summary>
    Public Function Update(fund As Fund, conn As SqlConnection,
                            trans As SqlTransaction) As Integer
    ' SQL: UPDATE M_FUND SET ... WHERE ISIN = @isin
    ' ※ISINは更新不可

    ''' <summary>論理削除（STATUS='9'に更新）</summary>
    Public Function SoftDelete(isin As String, updatedBy As String,
                                conn As SqlConnection,
                                trans As SqlTransaction) As Integer
    ' SQL: UPDATE M_FUND SET STATUS = '9', UPDATED_AT = GETDATE(), UPDATED_BY = @user
    '      WHERE ISIN = @isin

    ''' <summary>有効なファンド一覧（帳票出力のファンド選択用）</summary>
    Public Function GetActiveFunds(conn As SqlConnection) As List(Of Fund)
    ' SQL: SELECT * FROM M_FUND WHERE STATUS = '1' ORDER BY ISIN
End Class
```

**Save() の処理フロー：**

```
1. バリデーション
   - ISIN形式チェック（IsinValidator.Validate）
   - 新規時: ISIN重複チェック（FundRepository.Exists）
   - 通貨コード存在チェック（CurrencyRepository.Exists）
   - 籍国コード存在チェック（CountryRepository.Exists）
   - 設定日の未来日チェック
   → NG: BusinessException を Throw

2. トランザクション開始
3. 新規の場合:
   - FundRepository.Insert()
   - AuditLogRepository.Insert()
     → OperationType="INSERT", ColumnName=NULL（全カラム新規）
4. 更新の場合:
   - 更新前のFundを取得
   - FundRepository.Update()
   - 変更カラムを検出し、差分ごとにAuditLogを生成
   - AuditLogRepository.Insert()
5. トランザクションCommit
```

**監査ログ生成の詳細（UPDATE時）：**

```
OperationId = Guid.NewGuid().ToString()  ' 同一操作のグルーピング用
TableName = "M_FUND"
RecordKey = fund.Isin

変更検出対象カラム:
  FUND_NAME_EN, FUND_NAME_JP, CURRENCY_CODE, DOMICILE_COUNTRY_CODE,
  INCEPTION_DATE, SETTLEMENT_FREQUENCY, ROUNDING_TYPE, STATUS, REMARKS

各カラムについて oldValue <> newValue なら1レコード生成
```

| ImportExchangeRate(rows As List(Of String()), importLogId As Integer, conn As SqlConnection, trans As SqlTransaction) | 為替レート取込サブ |
| ImportCalendar(rows As List(Of String()), importLogId As Integer, conn As SqlConnection, trans As SqlTransaction) | カレンダー取込サブ |

**Import() の処理フロー（共通）：**

```
1. 共通前提チェック（CsvHelper.ValidateFile: ファイル存在、拡張子、サイズ、文字コード）
   → NG: BusinessException

2. CSV読込（CsvHelper.ReadCsv）→ rows取得
   ヘッダー行チェック（カラム名完全一致）
   → NG: BusinessException

3. T_IMPORT_LOG をINSERT（RUNNING状態）→ importLogId 取得

4. 全行の形式チェック（行単位バリデーション）
   → 1件でもエラーあり: 全エラーを T_IMPORT_LOG_DETAIL に記録
                        T_IMPORT_LOG を ERROR に更新
                        1件もDB登録しない（オールオアナッシング）
                        return ImportLog

5. 業務チェック（重複チェック等）
   → エラー: 同上
   → 警告: T_IMPORT_LOG_DETAIL に記録、取込は続行

6. トランザクション開始
7. 一括INSERT（NavHistoryRepository.BulkInsert 等）
8. トランザクションCommit

9. T_IMPORT_LOG を SUCCESS に更新
10. return ImportLog
```

**NAVデータ取込の業務チェック詳細（BD-006 §3.3準拠）：**

```
形式チェック（Validator実行、1行でもNGなら全件中止）:
  - ISIN: 12桁 + チェックディジット（Luhn）
  - NAVDate: 有効な日付 + 未来日不可
  - Currency: 3桁の英大文字であること（パターンチェックのみ）
  - NAVPerUnit: 正の数値 + 小数6桁以内
  - TotalNetAssets: 正の数値
  - SharesOutstanding: 正の数値

業務チェック（Service実行、DB参照を伴う）:
  - CurrencyがM_CURRENCYに存在: エラー → 全件中止
  - ISIN+NAVDate重複: エラー → 全件中止
  - ISINがM_FUNDに有効(STATUS='1')で存在: エラー → 全件中止
  - CurrencyがM_FUNDの通貨と一致: エラー → 全件中止
  - NAV変動率±10%超過: 警告 → 続行
```

### 5.5 NavCalculationService

バッチ処理の核。SCR-008（手動実行）とBatch/Program.vb（コンソール実行）の両方から呼び出される。

```vb
Public Class NavCalculationService
    Private ReadOnly _navRepo As NavHistoryRepository
    Private ReadOnly _rateRepo As ExchangeRateRepository
    Private ReadOnly _calRepo As CalendarRepository
    Private ReadOnly _logger As NLog.Logger
```

| メソッド | 概要 |
|---------|------|
| Execute(targetDate As Date) As BatchResult | バッチメイン処理 |
| GetTtmWithFallback(currencyCode As String, targetDate As Date, conn As SqlConnection) As RateLookupResult | 直前営業日レート遡及取得 |

**Execute() の処理フロー（BD-004 §2.3準拠）：**

```
1. 未換算レコード取得
   NavHistoryRepository.GetUnconverted(targetDate)
   → 0件: INFO "No records to process", return BatchResult(Success, 0)

2. FOR EACH target As NavCalculationTarget
   2-1. RoundingType, CurrencyCode は NavCalculationTarget に含まれる（JOIN済み）
   2-2. TTMレート取得（直前営業日遡及ロジック付き）
        GetTtmWithFallback(target.CurrencyCode, targetDate, conn)
        → 取得不可: ERROR記録、SKIP、CONTINUE
   2-3. 円換算計算
        navJpy = RoundingHelper.ApplyRounding(target.NavPerUnit * ttm, target.RoundingType)
   2-4. 個別トランザクションでUPDATE
        NavHistoryRepository.UpdateNavJpy(target.NavHistoryId, navJpy, ttm, DateTime.Now, AppConfig.UserName, conn, trans)
        → 成功: INFO記録
        → 失敗: ERROR記録、CONTINUE
   END FOR EACH

3. BatchResult（Total, Success, Skipped, Error, Elapsed, 明細リスト）を返却
```

**GetTtmWithFallback() の処理フロー（BD-004 §2.5準拠）：**

```
1. ExchangeRateRepository.GetRate(currencyCode, targetDate)
   → 取得できた: return RateLookupResult(rate, targetDate, isSubstitute=False)

2. lookbackCount = 0, currentDate = targetDate
3. WHILE lookbackCount < AppConfig.MaxRateLookbackDays (既定=5)
     currentDate = currentDate.AddDays(-1)
     IF 土曜 or 日曜 THEN CONTINUE（カウントしない）
     IF CalendarRepository.IsHoliday("JP", currentDate) THEN CONTINUE（カウントしない）
     lookbackCount += 1
     rate = ExchangeRateRepository.GetRate(currencyCode, currentDate)
     IF rate IsNot Nothing THEN
       WARN "Previous business day rate used: {currentDate}"
       return RateLookupResult(rate, currentDate, isSubstitute=True)

4. return Nothing（5営業日以内にレートなし）
```

## 6. Validators 詳細

**設計方針：** Validatorクラスは純粋な形式チェック（文字列長、パターン、数値範囲）のみを行い、DB参照は行わない。BD-006で「形式チェック」に分類されているがDB参照を伴う項目（CurrencyCodeのM_CURRENCY存在チェック、ISINのM_FUND存在チェック等）は、DataImportServiceの業務チェックフェーズで実施する。

### 6.1 IsinValidator

ISINコードのLuhnアルゴリズムによるチェックディジット検証。

```vb
Public Class IsinValidator
    ''' <summary>ISIN形式チェック（12桁 + チェックディジット）</summary>
    Public Shared Function Validate(isin As String) As Boolean
End Class
```

**Luhnアルゴリズム手順：**

```
1. ISINの先頭2桁（国コード）のアルファベットを数値に変換（A=10, B=11, ..., Z=35）
2. 3桁目〜11桁目の英数字も同様に変換（数字はそのまま、アルファベットは10〜35）
3. 変換後の数字列に対してLuhnチェックを実施
4. (10 - (sum mod 10)) mod 10 = 12桁目のチェックディジット
```
